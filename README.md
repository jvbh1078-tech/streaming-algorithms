# 스트리밍 알고리즘 구현 및 분석

> 데이터 엔지니어링 과제 — Bloom Filter & Count-Min Sketch  
> 동아대학교 | 학번: 2353936

## 개요

대용량 데이터 스트림 환경에서 전체 데이터를 저장하지 않고 근사 계산을 수행하는 스트리밍 알고리즘을 구현하고, 정확도·메모리·처리시간 측면에서 비교 분석한 실험입니다.

## 구현 알고리즘

| 알고리즘 | 목적 | 파일 |
|---|---|---|
| Bloom Filter | 원소 포함 여부 근사 판정 | `bloom_filter.py` |
| Count-Min Sketch | 항목별 빈도 근사 추정 | `count_min_sketch.py` |

## 데이터셋

MovieLens 1M 스타일 합성 스트리밍 데이터

- 레코드 수: 1,000,000

## 구현 특징

- 외부 라이브러리 없이 Python 표준 라이브러리(`hashlib`, `bytearray`)만 사용
- 해시 함수: MD5 + seed 기반 k개 독립 해시 생성
- 스트림 처리: 제너레이터 패턴으로 메모리에 전체 데이터를 올리지 않음
